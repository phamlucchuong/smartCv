# SmartCV - Frontend Monorepo

SmartCV is a job-matching platform that connects candidates and recruiters. This repository contains all three web applications and their shared packages, managed as a pnpm monorepo.

Backend repository: [smartCv-be](https://github.com/phamlucchuong/smartCv-be)

## Technical Stack

| Layer | Technology |
|---|---|
| Package manager | pnpm 9+ with Workspaces |
| UI framework | React 19 |
| Build tool | Vite 8 |
| Routing | TanStack Router v1 (file-based, fully type-safe) |
| Server state | TanStack Query v5 |
| Client state | Zustand v5 |
| Styling | Tailwind CSS v4 |
| Component library | shadcn/ui (via `@smart-cv/ui`) |
| Localization | i18next + react-i18next (via `@smart-cv/i18n`) |
| API codegen | Orval 7 — generates React Query hooks from live OpenAPI specs |
| HTTP client | Axios |
| Language | TypeScript 6 |
| Testing | Vitest 4 + jsdom |
| Linting | ESLint 10 (flat config) |

## Repository Structure

```
smartCv-fe/
├── apps/
│   ├── web-candidate/     # Candidate portal — job search, CV builder, applications (port 3000)
│   ├── web-recruiter/     # Recruiter portal — job posting, candidate management (port 3001)
│   └── web-admin/         # Admin dashboard — user and platform management (port 3003)
│
├── packages/
│   ├── api/               # Generated React Query hooks (@smart-cv/api)
│   ├── ui/                # Design system: shadcn/ui + Tailwind (@smart-cv/ui)
│   └── i18n/              # Shared localization, EN/VI JSON files (@smart-cv/i18n)
│
├── .env.example           # Environment variable template
├── package.json           # Root scripts
├── pnpm-workspace.yaml    # Workspace configuration
└── pnpm-lock.yaml         # Lockfile
```

Each app under `apps/` uses file-based routing. Route files live in `src/routes/`. The `routeTree.gen.ts` file is auto-generated by the TanStack Router Vite plugin — never edit it manually.

## Setup

### 1. Install dependencies

Run from the repository root. pnpm installs all workspace packages and links local packages together automatically.

```bash
pnpm install
```

### 2. Configure environment variables

Copy the root template into each application directory:

```bash
cp .env.example apps/web-candidate/.env
cp .env.example apps/web-recruiter/.env
cp .env.example apps/web-admin/.env
```

Open each `.env` file and set `VITE_API_BASE_URL` to the address of your running API Gateway (default: `http://localhost:8080`).

### 3. Generate the API client

The `@smart-cv/api` package generates typed React Query hooks by fetching the OpenAPI spec directly from each running backend service. The backend must be running before this step.

```bash
pnpm generate:api
```

See [API Client Generation](#api-client-generation) for details.

## API Client Generation

The `@smart-cv/api` package uses [Orval](https://orval.dev) to generate TypeScript types and TanStack Query hooks from the live OpenAPI specs exposed by the backend services.

### How it works

When any backend service runs, it exposes its OpenAPI spec at `/v3/api-docs`. Orval fetches these specs and generates typed hooks into `packages/api/src/generated/`.

| Service | Default spec URL |
|---|---|
| user-service | `http://localhost:8081/user/v3/api-docs` |
| job-service | `http://localhost:8082/job/v3/api-docs` |
| application-service | `http://localhost:8083/application/v3/api-docs` |
| ai-service | `http://localhost:8085/ai/v3/api-docs` |

Generated output is organized per service to avoid model conflicts:

```
packages/api/src/generated/
├── user/          # Auth, Users, Candidates, Companies, Settings, CV management
├── job/           # Jobs, Home aggregates, Related jobs
├── application/   # Applications
└── ai/            # AI recommendations
```

All hooks and types are re-exported from `@smart-cv/api`, so import paths in application code remain unchanged after regeneration.

### Running codegen

Regenerate hooks from all services (requires all backend services to be running):

```bash
pnpm generate:api
```

Regenerate from a single service only:

```bash
pnpm -F @smart-cv/api generate:user
pnpm -F @smart-cv/api generate:job
pnpm -F @smart-cv/api generate:application
pnpm -F @smart-cv/api generate:ai
```

### Overriding service URLs

Create a `.env` file in `packages/api/` based on `packages/api/.env.example`:

```bash
USER_SERVICE_URL=http://localhost:8081/user/v3/api-docs
JOB_SERVICE_URL=http://localhost:8082/job/v3/api-docs
APP_SERVICE_URL=http://localhost:8083/application/v3/api-docs
AI_SERVICE_URL=http://localhost:8085/ai/v3/api-docs
```

### Using generated hooks

```typescript
import { useGetActiveJobs, useSubmitApplication } from '@smart-cv/api';

const { data, isLoading } = useGetActiveJobs({ page: 1, size: 10 });
```

## Environment Variables

All Vite environment variables must be prefixed with `VITE_` to be accessible in the browser bundle.

| Variable | Default | Description |
|---|---|---|
| `VITE_API_BASE_URL` | `https://api.example.com` | API Gateway base URL used by the Axios instance |
| `VITE_I18N_DEFAULT_LOCALE` | `en` | Default locale on first visit (`en` or `vi`) |
| `VITE_I18N_FALLBACK_LOCALE` | `en` | Fallback locale when a translation key is missing |
| `VITE_WEB_CANDIDATE_PORT` | `3000` | Dev server port for web-candidate |
| `VITE_WEB_RECRUITER_PORT` | `3001` | Dev server port for web-recruiter |
| `VITE_WEB_ADMIN_PORT` | `3003` | Dev server port for web-admin |

| Command | Description |
|---|---|

| `pnpm typecheck` | Type-check all apps |
| `pnpm typecheck:candidate` | Type-check web-candidate only |
| `pnpm test` | Run all tests |
| `pnpm test:coverage` | Run all tests with coverage |

|---|---|

| Command | Description |
|---|---|
| `pnpm ci` | Full pipeline: install + generate:api + verify |
